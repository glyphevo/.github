# Contributing to GlyphEvo Labs

Thank you for your interest in contributing to GlyphEvo Labs projects.

This is the default contribution guide for repositories under the `glyphevo` organization. Individual repositories may provide more specific instructions when needed.

## Ways to Contribute

You can contribute by:

- Reporting reproducible bugs.
- Proposing focused feature requests.
- Improving documentation and examples.
- Submitting pull requests with tests where applicable.
- Sharing practical workflows related to AI agents, code intelligence, and context engineering.

## Issues

Before opening an issue, please check whether a similar one already exists.

A good issue includes:

- Expected behavior.
- Actual behavior.
- Steps to reproduce.
- Environment details.
- Relevant logs or screenshots when useful.

## Pull Requests

Before submitting a pull request:

1. Keep the change focused.
2. Explain the motivation and trade-offs.
3. Add or update tests when behavior changes.
4. Update documentation when user-facing behavior changes.
5. Avoid unrelated formatting churn.

## Branch Naming

Recommended prefixes:

- `feat/` for features.
- `fix/` for bug fixes.
- `docs/` for documentation.
- `refactor/` for internal restructuring.
- `test/` for test changes.
- `chore/` for maintenance.

## Commit Style

Recommended examples:

```text
feat: add repository graph extractor
fix: handle empty workspace
docs: update quick start
Project Status

GlyphEvo Labs projects are early-stage. APIs and repository structures may change while the projects evolve toward stable releases.
```
