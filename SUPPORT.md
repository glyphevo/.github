# Support

This document describes the default support channels for GlyphEvo Labs repositories.

## Questions

Use the repository's question issue template or discussion area if enabled.

A good question includes:

- What you are trying to build.
- What command, API, or workflow you used.
- What result you expected.
- What happened instead.
- Relevant logs or screenshots when useful.

## Bugs

Use the bug report issue template in the affected repository.

Please include reproducible steps, environment details, and minimal examples when possible.

## Feature Requests

Use the feature request template. Explain:

- The problem or workflow pain point.
- Why the feature belongs in the project.
- Alternatives you considered.
- Any compatibility or security concerns.

## Security Issues

Do not report vulnerabilities in public issues. Follow `SECURITY.md`.

## Commercial Support

GlyphEvo Labs does not currently provide a formal SLA or commercial support plan. This may change as projects mature.
